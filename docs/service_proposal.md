## 2. 주요 기능

### 2.1 사용자 맞춤형 검색
- 거주지역(시/군/구), 연령, 생애주기 정보 기반 자동 필터링
- 사용자 프로필 저장으로 재방문 시 빠른 서비스 제공

### 2.2 AI 기반 검색 엔진
- **벡터 검색**: OpenAI text-embedding-3-small 모델 사용
- **의미 기반 검색**: "치매 관련 서비스" 같은 자연어 질의 지원
- **하이브리드 검색**: 규칙 기반 필터링 + AI 벡터 검색 결합

### 2.3 복지 서비스 정보
- 국가/지방 복지 서비스 통합 제공 (복지로 API 연동)
- 생애주기별, 대상별 맞춤 정보
- 신청 방법, 기간, 담당 기관 정보 제공

### 2.4 공공 일자리 정보 (개발 예정)
- 고용24 채용정보 API 연동
- 시니어 대상 공공일자리 필터링
- 지역별, 직종별 맞춤 추천

## 3. 기술 스택

### Backend
- **AWS Lambda**: 서버리스 아키텍처 (Python 3.11)
- **Supabase**: PostgreSQL + pgvector (벡터 검색 지원)
- **AWS SAM**: 인프라 관리 및 배포

### AI/ML
- **OpenAI Embeddings**: text-embedding-3-small (1536 차원)
- **Vector Search**: pgvector + HNSW 인덱스
- **RAG (Retrieval Augmented Generation)**: 검색 증강 생성

### Frontend
- **KakaoTalk Skill API**: 챗봇 인터페이스
- **Webview** (예정): 비즈니스 채널 승인 후 도입

### Data Pipeline
- 복지로 API (보건복지부)
- 고용24 채용정보 API (한국고용정보원)
- 자동 데이터 수집 및 임베딩 생성

## 8. 개발 증빙 자료

### GitHub Repository
- 프로젝트 구조: backend/ (Lambda functions), scripts/ (데이터 수집), supabase/ (DB schema)
- 주요 파일:
  - `backend/functions/kakao_webhook/app.py`: 챗봇 핸들러
  - `backend/common/rag_service.py`: AI 검색 엔진
  - `scripts/embeddings/generate_embeddings.py`: 임베딩 생성
  - `supabase/schema.sql`: 데이터베이스 스키마

### AWS 배포 현황
- Lambda Functions: 4개 (kakao_webhook, data_collector, region_updater, keep_alive)
- API Gateway: REST API 엔드포인트
- 배포 도구: AWS SAM

### 데이터베이스
- Supabase PostgreSQL
- 테이블: benefits (복지 서비스), benefit_embeddings (벡터 검색), users (사용자 프로필)
